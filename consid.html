Conclusiones finales

Como primera conclusión, se puede afirmar que el seguimiento de las instrucciones detalladas en este proyecto, permite la creación de un mini servidor de bajo coste y mínimo mantenimiento y ofrece al administrador del sistema, un servidor web básico y con un entorno estable para la instalación de aplicaciones web.

Se ha registrado un dominio muy económico, exclusivamente para la presentación y desarrollo de este proyecto, permitiendo así que cualquier usuario pueda acceder en todo momento. El proyecto también se ha publicado en GitHub para que los alumnos interesados puedan realizar modificaciones, o utilizarlo para iniciar nuevos trabajos. Además, el proyecto ha sido publicado bajo la licencia de Creative Commons Reconocimiento 4.0 Internacional.

  • Nombre del dominio: www.raspberryasir.es
  • Enlace a GitHub: https://github.com/Nacholoop/enigma

A medida que ha ido avanzando y desarrollándose el proyecto, a pesar de haber superado considerablemente el número de horas destinadas a su elaboración tan solo se ha podido concluir una parte de la idea inicial. Se ha dedicado gran parte del tiempo únicamente en comprobar y actualizar todo el material encontrado en la web, (obsoleto en gran medida), aunque se trata de una parte fundamental para la estructuración de un trabajo de calidad.

Se deja abierta una puerta que permite profundizar en el uso de Docker o mostrar más aspectos sobre la seguridad en la informática, ya que una de las funciones prácticas pasa por que los alumnos utilicen las herramientas de seguridad y puedan lanzar ataques para probar las defensas incluidas en el servidor.

Para finalizar se plantean posibles proyectos a partir de éste:

  • Realizar todos los pasos de este proyecto en otro sistema operativo como Ubuntu, puesto que es el sistema usado en el ciclo de Asir (Ubuntu 18.04 LTS).
  • Instalación y configuración de Ldap para la autenticación de los alumnos, así como un servidor radius para controlar esas mismas cuentas por wifi (free radius,open ldap).
  • Instalación de Samba y servidor NAS.
  • Ansible: Combina instalación multi-nodo, ejecuciones de tareas ad hoc y administración de configuraciones, para dejar operativo el servidor con un solo paso.
